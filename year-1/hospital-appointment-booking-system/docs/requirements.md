# Requirements (Summary)

## Functional requirements
- FR1: The system shall allow registering a patient with (ID, name, phone).
- FR2: The system shall allow adding a doctor with (ID, name, specialty).
- FR3: The system shall allow booking an appointment for (patient, doctor, datetime).
- FR4: The system shall prevent double-booking the same doctor at the same datetime.
- FR5: The system shall allow cancelling an appointment.
- FR6: The system shall display a doctor's schedule.
- FR7: The system shall display a patient's appointments.
- FR8: The system shall save and load data using files.

## Non-functional requirements
- NFR1: Simple console UI with clear prompts and validation.
- NFR2: Data persistence via CSV files for portability.
- NFR3: Modular code structure (models, storage, app logic).
